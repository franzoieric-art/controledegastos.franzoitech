Guia Completo de Comandos Git

Uma referência interativa para os comandos mais úteis do Git no seu dia a dia.

1. Comandos Essenciais para Iniciar um Projeto

Esta seção cobre os primeiros passos indispensáveis para transformar uma pasta local em um repositório Git e enviar seu projeto para o GitHub. São os comandos fundamentais que formam a base para todo o controle de versão.

git init

Para que serve: Inicia um novo repositório Git na sua pasta local.

git add .

Para que serve: Adiciona todos os arquivos modificados para a área de "stage", preparando-os para o próximo commit.

git commit -m "Sua mensagem"

Para que serve: Salva as mudanças que estão na área de "stage" no seu histórico local do Git.

git remote add origin <URL>

Para que serve: Conecta o seu repositório local a um repositório remoto no GitHub.

git push -u origin main

Para que serve: Envia as suas alterações locais para o GitHub pela primeira vez.

2. Comandos para Fluxo de Trabalho e Branches

Depois que seu projeto está configurado, estes comandos se tornam parte da sua rotina. Eles permitem que você gerencie diferentes linhas de desenvolvimento (branches), sincronize com o repositório remoto e lide com alterações de forma segura e organizada.

git branch -a

Para que serve: Lista todas as branches do seu projeto, tanto as locais quanto as remotas.

git fetch

Para que serve: Baixa as informações mais recentes do repositório remoto sem alterar seus arquivos locais.

git checkout <nome-da-branch>

Para que serve: Troca para uma branch existente no seu projeto.

git checkout -b <nova-branch>

Para que serve: Cria uma nova branch e já troca para ela em um único comando.

git stash

Para que serve: Guarda temporariamente as alterações não salvas, permitindo que você troque de branch sem perder o trabalho.

git stash pop

Para que serve: Aplica de volta as mudanças que foram guardadas com git stash.

3. Dicas e Soluções de Problemas

Todo desenvolvedor encontra pequenos desafios ao usar o Git. Esta seção oferece soluções para problemas comuns e dicas para otimizar seu ambiente de trabalho, como ignorar arquivos desnecessários e gerenciar múltiplos projetos de forma eficiente no VS Code.

Como lidar com arquivos temporários como o .DS_Store

Para evitar que o Git rastreie arquivos de sistema ou pastas de dependências (como `node_modules`), use um arquivo `.gitignore`.

  1. Crie um arquivo chamado .gitignore na pasta principal do seu projeto.
  2. Dentro dele, adicione o nome do arquivo ou pasta que você quer ignorar. Por exemplo:
.DS_Store
node_modules/
.env

Salve o arquivo. O Git não vai mais se preocupar com eles. Lembre-se de fazer um commit do arquivo `.gitignore`!

Como trabalhar com vários projetos na mesma janela

Para gerenciar múltiplos projetos relacionados sem ter que abrir várias janelas do VS Code, utilize o recurso de Workspace.

  1. Com um projeto já aberto, vá em File > Add Folder to Workspace...
  2. Selecione a outra pasta de projeto que deseja adicionar.
  3. Repita para quantos projetos precisar.
  4. Vá em File > Save Workspace As... para salvar a configuração.

Agora, basta abrir o arquivo `.code-workspace` para carregar todos os seus projetos de uma vez, cada um com seu próprio controle Git independente.
